# bd チケット運用規約

## 計画は bd チケットに登録する

計画した内容は bd チケットに作成し、実装完了したらチケットをクローズする。

1. Plan mode で計画を作成
2. 計画の各ステップを bd チケットとして登録 (`bd create`)
3. 実装時にチケットを claim (`bd update <id> --claim`)
4. 完了したらチケットをクローズ (`bd close <id>`)

## /ralph-tornado 実行時のチケット必須化

/ralph-tornado で plan 実行する際は、必ず bd チケットを起票し、計画全体の内容を description に記載すること。完了時にクローズする。チケットなしでの実行は禁止。

## 結合テストでバグ発見時は bd チケットを起票

結合テスト（`just mock` / `just live`）でバグが発見された場合、各バグについて bd チケット（type=bug）を個別に起票する。

バグをその場で直すのではなく、チケットとして追跡可能にすることで、作業の可視性と優先度管理を確保する。
