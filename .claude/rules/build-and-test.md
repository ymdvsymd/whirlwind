# ビルド・テスト規約

## just コマンドを使うこと

ビルド・テスト等には必ず `just` コマンドを使う。`npm run` や `moon test` を直接叩かない。

justfile がプロジェクトの正規ワークフローを定義しており、SDK ビルド等の依存関係も管理している。個別コマンドを直接叩くと依存の実行漏れが起きる。

- テスト: `just test`
- 型チェック: `just check`
- ビルド: `just build`
- 実行: `just run`
- 整形: `just fmt`

## live 結合テストの実行は必須

テストの検証には結合テスト (live) の実行が必須。unit test (`just test`) だけで終わらせない。

`just test` に加えて `just live` も実行すること。mock テストだけでは実エージェント環境での問題を検出できない。

## バグ修正にはリグレッションテスト必須

バグ修正では、同じバグが再発しないことを保証する自動リグレッションテストを必ず追加すること。

修正前に失敗し修正後に成功するテストを必ず作成する。テストなしでバグ修正をコミットしない。
