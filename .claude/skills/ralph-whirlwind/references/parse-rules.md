# Markdown パース規則

## パース規則テーブル

| Markdown 要素                          | 変換先                                                |
| -------------------------------------- | ----------------------------------------------------- |
| `# 見出し` (最初の H1)                | プロジェクト名（ディレクトリ名に使用）                |
| `#` と最初の `##` の間のテキスト       | `brief` に統合                                        |
| `##` 情報セクション（セマンティック一致） | `brief` に統合（マイルストーンにしない）              |
| `##` 見出し（上記以外）                | マイルストーンの `goal`（後述の goal 構成規則に従う） |
| `##` がない場合                        | ファイル全体を 1 マイルストーンとして扱う             |

## 情報セクションの判定規則

以下のセマンティックカテゴリに該当する `##` セクションは `brief` に統合する。
見出しの正確な文言ではなく**意味**で判断する:

| カテゴリ                                                     | 例                                                         |
| ------------------------------------------------------------ | ---------------------------------------------------------- |
| 背景・文脈                                                   | Context, 背景, Background, 経緯, 前提, Why this change     |
| 変更対象ファイル一覧（実装詳細を含まない単なるリスト）       | 変更対象ファイル, 修正対象ファイル, Target Files            |
| 設計判断・方針                                               | Design Decisions, 設計方針, 互換方針, 設計判断             |
| 問題分析・原因調査                                           | 推定原因, 問題の分析, Root Cause, Analysis                 |
| 調査結果・結論                                               | 結論, 調査結果, Conclusion, Results                        |

上記に該当しない `##` セクションはすべてマイルストーンとする。
**迷った場合はマイルストーンにする**（情報欠落より冗長の方が安全）。

## goal の構成規則

各マイルストーンの `goal` は以下の構成で生成すること:

```
<## 見出しテキスト>

<見出し配下の本文全体（次の ## または EOF まで）>
```

- `goal` には `##` 見出しテキスト＋その配下の本文全体を含める
- 本文中のコード例、ファイルパス、行番号、手順説明、`###` サブセクション等すべて含む
- `goal` は英語で生成する
- コード内の識別子やファイルパスはそのまま保持する

## ディレクトリ名の生成

- 最初の `# 見出し` のテキストを英語に翻訳してタイトルとする
- 小文字化、スペース/アンダースコアをハイフンに変換、英数字とハイフン以外を除去、50文字以内に切り詰める
- 出力ディレクトリ: `{repo_root}/.runs/{yyyy-mm-dd_HHmm}_{kebab-case-name}/`
